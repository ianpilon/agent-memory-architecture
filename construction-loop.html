The construction process

Memory as a construction loop

The thesis is that memory is the construction process by which an agent persists across time. That process has four phases, and the same four run in a brain and in an engineered agent. Biological memory and AI memory differ at the substrate; they share the function. This page draws that shared function, and lines the two substrates up against it.

The four phases

A construction process, not a storage pipeline

1
Encode
function · gate by goals and salience
Nothing is stored blind. The stream is filtered by the agent's current goals and emotional weight before it settles. In a brain this is the working-self gate and somatic marking; in an agent it is salience-weighted writing into the base of the pyramid.
2
Consolidate
function · compress into a revisable hierarchy
A background process compresses episodes into a hierarchical, rewritable structure: from raw event up to lifetime themes. Conway calls it the autobiographical knowledge base; NapMem re-derives it as the memory pyramid. Either way, the structure can revise itself and forget.
3
Reconstruct
function · regrow for the present, never fetch
On the way out a memory is a seed regrown in context, not a record returned. Biology does this through the bowtie's creative decoder; engineering does it through active navigation of the pyramid, stopping when the evidence is enough. Retrieval is generation.
4
Synthesize
function · weave memory into an evolving self
The phase no storage system has. Reconstructed memories are woven into a narrative self that co-emerges with what is remembered. In biology that is coherence and the autobiographical self; in engineering it is the proactive agent's selective intervention, where the decision to stay silent is what shapes identity over time.
The core argument

One function, two substrates

Read each row across. The middle column is the function. The left column is how a brain implements it; the right column is how an engineered agent can. They are built from different stuff, but they are doing the same thing, and that is the whole basis for calling both of them memory.

Functionally homologous, materially different

This is the sentence the whole thesis rests on. Biological memory and AI memory are the same process (a construction that maintains identity across change) built on different stuff, with different fidelity, different forgetting dynamics, and different ceilings on the substrate-flexibility layer. The name "memory" lives at the function column. The substrate difference is real and it is the thing the open questions are about, but it is not a reason to deny that both columns are memory.

The substrate is not fixed either

What the polycomputing source adds

For most of the table above, the engineered column rests on ordinary substrates: weights and JSONL stores. The honest worry was whether Levin's layer, genuine substrate flexibility, was reachable in engineered systems at all, or whether it was a biological property silicon could only imitate at the function level.

Bongard & Levin's polycomputing paper closes that worry. One substrate can simultaneously compute different things for different observers, and engineered materials already do this: polycomputing metamaterials, physical reservoir computing, holographic storage, memory materials. The substrate is not a fixed slot a value is parked in; it is an excitable medium a construction persists through. That is why the loop closes, and why the self at month 12 can differ from the self at month 1 without breaking.

One function. Two substrates. And the substrate itself turns out to be flexible.
