Reference architecture

A memory architecture where the agent persists, not just retrieves

Five engineering layers derived from Conway's psychology, Damasio's neuroscience, and Levin's biology, mapped to what you can ship today and what is still an open research problem.

Memory is not storage. It is the construction process by which a self maintains identity across change. Treating it as a database is the category error behind why agent memory falls apart past a few thousand documents.
The stack / five layers

The layers build on each other, bottom to top

Layers 1 to 4 are Youssef's five principles restated as buildable components. Layer 5 is the Levin substrate claim that his framework does not reach. Each carries a source lineage and a build verdict.

1

Hierarchy and goal-filtering

Conway · working-self + autobiographical KB

Do not flat-search every fragment at equal depth. Store memory as a partonomy (epochs, then recurring themes, then specific episodes) and let a live model of the agent's current goal gate which branch is even opened. The working self decides what is worth encoding and what may enter consciousness.

Build as
Nested temporal store (epoch → theme → episode) plus a goal-conditioned retrieval planner that rewrites the query before it hits the index. Temporal knowledge graphs (Zep / Graphiti) already approximate this. NapMem (Xu et al., 2026) is the cleanest worked example: its four-level pyramid (raw conversations → records → topic tracks → profile) independently re-derives Conway's autobiographical hierarchy from engineering pressure, and ships competitively.
Verdict
Shippable today
2

Active reinterpretation on retrieval

Levin · bowtie architecture

A retrieved memory is a compressed engram with no metadata attached. It cannot be handed back raw. The read side must creatively re-inflate it in the current context, with allegiance to who is asking now, not to who wrote it. Retrieval is generation, not a nearest-neighbor lookup.

Build as
Replace "return top-k chunks" with "retrieve, then have the model synthesize them against the present task." The mechanism exists (query-focused synthesis), and two 2026 papers now give it measured form. NapMem (Xu et al.) trains the action agent with GRPO over memory-tool trajectories and shows active navigation beats passive retrieval with the same sources. The Proactive Memory Agent (Wu et al., Meta) goes further: it runs a separate memory agent beside an unmodified action agent, and its ablation shows the win lives in the decision whether to intervene at all, beating passive full-bank exposure and always-on injection. Doing reinterpretation trustworthily over long horizons is still the hard part: the same move that produces useful reinterpretation also produces confabulation and drift.
Verdict
Buildable, recipe now published
3

Salience and emotional weighting

Damasio · somatic markers + Conway conservatism

Emotionally significant experience encodes deeper and returns faster. A frustrated exchange should not sit at the same weight as a routine query. Salience should bias three things: what gets stored at all, what surfaces first, and how the agent reads its own current state.

Build as
Valence-arousal scores as node metadata (classifiers are mature, this part is easy) wired into encoding depth, retrieval ranking, and a running affect signal. The open work is using the score to gate encoding and behavior, not just to boost ranking.
Verdict
Scoring trivial, wiring it in is not
4

Narrative layer and co-emergent self-model

Bruner + Klein/Nichols · narrative-coherence + co-emergent self

Episodes become a self only when a story binds them. The agent should hold "what story am I in with this user," and that self-model should update from experience so the agent is genuinely different at month twelve than at month one. This is where identity, not just recall, lives.

Build as
A periodic reflection pass that rewrites an explicit self-state object from accumulated memory (the pattern in Stanford's generative agents). Prototypes exist. What is unsolved: keeping the narrative coherent and load-bearing over long horizons without drift, sycophancy, or personality collapse.
Verdict
Research frontier
5

Substrate flexibility

Levin · polycomputing + no-single-substrate

The Levin layer, which Youssef's five principles do not touch. Memory as an active, agential pattern that can be re-read from a new perspective without being rewritten, distributed across a reservoir rather than parked in one store. The engram partly shapes the agent that holds it.

Build as
No clean pattern on today's stack. A vector index gives you the passive read side of polycomputing (one stored vector answers many queries). The active write side, memory that directs other subsystems, does not exist. Candidate substrates (reservoir computing, neuromorphic) are not integrated with agent frameworks.
Verdict
Blocked on substrate
Data flow

How a memory moves through the stack

The write path and read path reuse the same layers in different orders. The read path is the one that breaks the database metaphor: nothing is returned raw.

Write path

Encoding an interaction

  • Score salience of the incoming exchange. L3
  • Working-self gate decides whether and how deeply to encode it, given current goals. L1
  • Place in the partonomy: attach to the right epoch, theme, and episode. L1
  • Reflection pass periodically rewrites the self-model from what has accumulated. L4

Read path

Answering in context

  • Goal builds the query: retrieval is conditioned on the present task, not raw similarity. L1 / L2
  • Descend the hierarchy instead of flat-searching everything. L1
  • Re-inflate, do not return: synthesize the engram against the current moment. L2
  • Condition on self-model and salience, then let the output update the self. L3 / L4 / L5

This is why a plain vector store fails as identity memory: it treats the narrow waist as an address to fetch from, when the architecture says the waist is a seed that has to be regrown for whoever is asking now.

Deployment pattern: run memory as a separate agent

The two measured sources imply different shapes. NapMem trains one model to both navigate memory and answer. The Proactive Memory Agent keeps the action agent untouched and adds a separate memory agent that runs on a fixed interval, deciding whether to inject a reminder or stay silent. Its ablation shows that silence beats always-on injection and passive full-bank exposure.

The split is the better default for a real deployment. It lets you size the two models independently (a strong action agent, a cheaper memory sidecar), it works against any frontier action agent you cannot retrain, and the cost lands on a bounded schedule rather than on every query. Build the pyramid as the store (L1), then put a selective intervention agent on top of it (L2).

The verdict / what is possible today

What you can ship, what needs research, what the substrate blocks

The honest reading of the five layers against 2026 tooling. The line between buildable and blocked runs right through Youssef's fifth principle and into Levin's territory.

Bottom line

Four fifths of this is an engineering project. The last fifth is a research bet.

Build this now

Layers 1 through 3 are a real product. A hierarchical, goal-gated, salience-weighted memory that reconstructs on retrieval instead of fetching is assemblable from mature parts today. It would already beat flat vector recall on the failure mode Youssef names: coherence past ten thousand documents. Layers 1 and 2 now have measured engineering evidence behind them in NapMem.

The catch on Layer 2 and the harder half of Layer 3 is quality, not feasibility. You can wire it; making reinterpretation trustworthy is the tuning problem.

Treat this as research

Layer 4 is where identity actually lives, and no one has shipped it well. Reflection loops give you a narrative layer in prototype form, but a self-model that stays coherent and grows over a year without drifting into sycophancy is an open problem the frontier labs have deliberately not solved in production.

Do not block the first three layers on it. Ship them, instrument drift, then reach for the narrative layer.

The Levin layer is the honest unknown

Layer 5 is not "hard to build," it is "unclear the material can hold it." Levin's claim is that memory is an active, agential, substrate-independent pattern. On an LLM plus a vector database, memory is passive data you look up. You can imitate the read side of polycomputing, but the write side (an engram that reshapes the agent that carries it) has no home on this substrate.

NapMem sharpens this question without answering it. It shows the functional layer (hierarchy plus active reinterpretation) delivers competitive results on a perfectly ordinary substrate, so the question is no longer "can we build good agent memory" but "is there a ceiling the ordinary substrate imposes that only genuine substrate flexibility can break?"

The Proactive Memory Agent sharpens it further by isolating what actually carries the gains: not the store, and not even active reinterpretation alone, but the selective decision to intervene. Its ablation shows the silence action beats exposing the full bank every step. So the floor is set less by the substrate than by whether the control architecture can decide when memory should enter the loop. That control layer, not the substrate, may be the real ceiling to test.

The open question the wiki flags directly: can any engineered substrate support the Levin layer at all, or does the LLM-plus-index floor cap how much identity-coherence you can ever reach? Reservoir computing and neuromorphic hardware are the named candidates, and neither is wired into an agent stack yet.
